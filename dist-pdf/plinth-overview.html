The 60-second story

What Plynf is, in plain English.

Imagine you hired a freelance researcher to write you a report. Two ways to manage them.

Way A

The phone call

You keep them on a phone call the whole time. Every time they want to remember something, they read it back to you aloud. Every new source, they read aloud. By hour three the call is enormous, every minute costs a fortune, and they've half-forgotten what they decided in hour one.

→ This is how most AI agents work today.

Way B

The desk

You give them a desk. They put notes down, pick them up, file research, label folders, bookmark pages. When they need something, they look it up. The desk has drawers. The drawers have labels. Three hours in, the desk has structure, and they remember what they decided in hour one because it is written on the desk.

→ This is Plynf.

Plynf is the desk for AI agents. It is the part of the system that holds onto what the agent has learned, fetched, and decided — so the agent doesn't have to keep saying it out loud, over and over, to itself.

The phone-call problem is not theoretical. A naive 10-step research workflow sends the entire prior conversation back to the model with every step. By step 10 you're paying ten times what step one cost, every time. The bill scales with the square of the work — which is why most "production agents" are actually three-step demos that fall over the moment you ask them to do something serious.

Plynf replaces the phone-call architecture with the desk architecture. Same agent, same model, same output. About 71% fewer tokens. Measured, not marketed.

Plynf · Overview 02

What's in the runtime

Three primitives. One coherent surface.

Each one removes a category of plumbing every agent team rebuilds. They are designed together, not retrofitted.

01 · Persistent workspace

Key-value store, files, snapshots, branches. Every write is versioned. The agent's memory survives crashes, restarts, and 50-step workflows. Take a labelled checkpoint before a risky tool call; restore in one line if it goes wrong.

02 · Tool gateway

Single boundary for every external tool. OAuth, caching, audit log, cost caps. The agent calls tools.invoke("web.search", …) and never sees a credential. Eight providers built-in: GitHub, Slack, Linear, Notion, Google, Atlassian, Salesforce, Asana.

03 · Multi-agent channels

Durable typed message queues between specialist agents. Researcher writes drafts; Writer reads them and composes; Reviewer reads and approves. Each agent keeps a narrow prompt. No agent has to know what the others are doing.

The mechanism in one sentence

Instead of re-sending the entire prior conversation with every model call, the agent gets back handles — short strings like ws://my-research/sources/renewables.txt@v17 — and dereferences them only when needed, in slices.

Plynf · Overview 04

For developers

One SDK call. The runtime does the rest.

The same primitives in Python, TypeScript, Go, Swift, and Kotlin. API v1 is locked: additive-only for at least 12 months.

# Python · the canonical SDK from plinth import Plynf client = Plynf( workspace_url="http://localhost:7421", gateway_url="http://localhost:7422", api_key="local-dev", ) ws = client.workspace("my-research") ws.kv.set("topic", "renewable energy") # Stored in workspace, doesn't bloat prompt history sources = client.tools.invoke("web.search", {"query": "renewable energy"}) for src in sources["results"]: content = client.tools.invoke("web.fetch", {"url": src["url"]}) # cached ws.files.write(f"sources/{slug(src['url'])}.txt", content) ws.snapshot("sources-collected") # named checkpoint

What the SDK gives you

  • Handles, not values. Reads return URIs; the model only sees content on explicit dereference.
  • Auto-projections. Summaries, vector indexes, and outlines generated on write — async, no blocking.
  • Branch & merge. Fork the workspace, try three approaches in parallel, keep the winner.
  • Lease-based workers. Crash mid-flight, pick up where you left off. Verified by a runnable demo.

The contract you get

  • API v1 is locked. Additive-only for 12+ months. Breaking changes go to a /v2/ namespace with migration guide.
  • Apache 2.0. The Cloud product runs identical code. No "community edition" with the good parts ripped out.
  • 2,867 tests green. Three SDKs in Python/TS/Go to API parity; Swift and Kotlin to feature parity in v2.
  • Threat model published. 90-day fix window on vulnerabilities. Bounty matrix on /security.
